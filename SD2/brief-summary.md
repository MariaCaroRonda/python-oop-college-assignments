# Brief Summary: SD2 ATM Program

## Module

Fundamentals of Object-Oriented Programming 5N0541

## Assignment Title

Skills Demonstration 2: Design, code, and test an object-oriented program that simulates a bank's Automatic Teller Machine (ATM).

## Task Summary

The task was to create a single-class, object-oriented Python program that simulates an ATM.

The program needed to support at least four users and allow each user to:

- Log in with a user ID and PIN
- Check their account balance
- Withdraw cash
- Lodge cash
- Exit the program

## Main Requirements

The assignment required:

- A simple algorithm
- A flowchart
- Pseudocode
- A data dictionary
- A working Python program
- A single-class object-oriented structure
- Appropriate variables and data types
- Clear comments
- Suitable variable and object names
- Testing and debugging evidence

## Implementation Summary

My solution uses an `ATM` class to store user account details and manage the main ATM actions.

The program stores multiple users in a dictionary, with each user having a PIN and account balance. After successful login, the user can check their balance, withdraw cash, lodge cash, or exit using a menu-driven interface.

The program includes validation for invalid menu options, invalid number input, negative or zero cash amounts, incorrect user IDs, incorrect PINs, and withdrawal amounts greater than the current balance.
